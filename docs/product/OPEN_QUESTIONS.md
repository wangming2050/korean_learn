# Korean Learn 产品待确认问题

更新日期：2026-06-22

这些问题会影响后续技术方案、数据模型和排期，建议在进入大功能开发前逐步确认。

## 1. 用户与商业边界

1. 首批目标是个人自用、小范围测试，还是公开用户？
2. 未来是否考虑付费功能？
3. 是否需要支持老师、课堂或机构使用？

## 2. 内容版权

1. 教材是否长期坚持用户自上传？
2. 是否会提供官方内置教材包？
3. 如果提供内置教材包，内容、音频和图片授权如何解决？
4. 当前词汇和句子数据是否需要标注来源？

## 3. AI 服务选择

1. 教材页视觉问答是否统一使用 Gemini 2.5 Flash？
2. DeepSeek 是否只用于纯文本问答？
3. 是否需要保留 OpenAI provider？
4. AI 历史记录是否需要跨端同步？
5. 用户上传 PDF 的截图发送到 AI 服务前，是否需要隐私提示？

## 4. 音频策略

1. 四十音本身是否必须替换为真人标准音？
2. 示例词是否也要真人音？
3. TTS 是否只作为兜底？
4. 音频文件最终放在 Git、Railway、R2/CDN，还是数据库只存 URL？

## 5. 账号和同步

1. 下一阶段是否优先做登录？
2. 登录方式用邮箱、手机号、Apple、微信，还是先用简单账号密码？
3. 同步范围包括哪些数据？
4. 用户 PDF 是否同步整本文件，还是只同步元数据和笔记？

## 6. 移动端技术路线

1. 是否继续使用 Expo/React Native？
2. 是否接受 iPhone 和 iPad 同一套代码，按屏幕适配？
3. 是否需要上架 App Store？
4. 是否先只做本地测试，不接 TestFlight？

## 7. 教材批注能力

1. 批注是直接画在 PDF 页图上，还是做独立图层？
2. 划线需要绑定文本，还是只绑定坐标？
3. 手写批注是否必须支持 Apple Pencil 压感？
4. 批注是否需要跨端同步？

## 8. 学习记录

1. 词汇掌握状态如何定义？
2. 音标练习是否记录次数？
3. 跟读评分是否进入学习记录？
4. 是否需要每日学习统计？

## 9. 国际化和多语言

1. 中文版本验证完成后，第一批多语言版本优先支持哪些语言？
2. 多语言优先级按用户规模、目标市场、学习需求，还是实现成本排序？
3. 多语言内容采用人工维护、AI 辅助生成后人工审核，还是 AI 实时翻译？
4. 音标发音解释是否需要针对不同母语用户单独编写？
5. 教材 AI 助教的回答语言是否默认跟随用户母语？
6. Web、iPhone、iPad 是否同时支持多语言，还是先 Web 后移动端？
7. 多语言版本是否需要保留中文作为默认兜底语言？
