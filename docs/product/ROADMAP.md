# Korean Learn 后续迭代路线图

更新日期：2026-06-23  
文档版本：v2，按账号基础、必补能力、创新点、长期规划重排

## 1. 路线图原则

后续迭代按以下优先级推进：

1. 先完成用户登录和用户数据归属。
2. 先补齐当前 Web 和移动端必须完善的基础能力。
3. 再推进创新点，包括教材 AI、语音对话、文本转语音、发音评分、音标和词汇联动。
4. 在中文版本验证完成后推进国际化和多语言版本，让产品面向全球不同母语用户。
5. 最后推进更长期的内容生态、复杂个性化和商业化能力。

核心判断：

没有登录态之前，不适合做真实学习计划、用户自导入内容、跨端同步、词汇状态、教材句子沉淀和个性化复习。  
所以账号和用户数据模型必须前置。

## 2. 当前版本基线

当前项目可视为：

- Web 主产品。
- 移动端原型。
- iPad 方向规划中。

当前 Web 已具备：

- 学习路径。
- 音标学习。
- 词汇浏览。
- 例句学习。
- 用户本地 PDF 教材阅读。
- 教材页 AI 问答基础能力。
- Google TTS 兜底能力。
- 管理后台的场景和句子维护。

当前移动端已具备或正在建设：

- Expo/React Native 通用 App。
- 通过 Web API 拉取学习数据。
- iPhone/iPad 共用一套代码。
- 音标、词汇、教材等页面的 UI 和数据对齐。

当前主要缺口：

- 没有用户登录。
- 没有用户数据归属。
- 没有跨端同步。
- 音标、词汇、例句、教材之间的数据联动不足。
- 用户自导入内容还没有统一数据模型。
- 语音评分、AI 语音对话、教材文本转语音仍未实现。
- 多语言 / 国际化能力未实现，当前仍以中文用户验证产品闭环。

## 3. 阶段一：用户登录与数据基础

优先级：P0  
阶段目标：让用户数据有归属，为后续所有创新点打地基。

### 3.1 为什么必须优先做

以下能力都依赖用户身份：

- 用户自己添加关联词汇。
- 用户导入词汇。
- 用户导入例句。
- 教材中的句子添加到例句库。
- 用户教材库。
- 阅读进度。
- 词汇状态。
- 音标练习记录。
- 语音评分历史。
- AI 语音对话记录。
- 跨 Web、iPhone、iPad 同步。

如果没有用户登录，这些能力只能做成本地临时状态，无法成为真实产品能力。

### 3.2 建议能力

账号：

- 用户注册。
- 用户登录。
- 用户退出。
- 登录状态保持。
- 基础用户资料。

数据归属：

- 所有用户自导入内容必须绑定用户。
- 所有学习记录必须绑定用户。
- 所有教材、笔记、收藏、历史必须绑定用户。

基础同步：

- Web 和移动端使用同一套用户身份。
- 登录后读取用户数据。
- 未登录用户仍可使用基础浏览能力，但不能保存跨端数据。

### 3.3 建议新增数据模型

- 用户表。
- 用户会话表。
- 用户教材表。
- 用户词汇表。
- 用户例句表。
- 用户音标练习记录表。
- 用户词汇学习状态表。
- 用户句子练习记录表。
- 用户 AI 历史表。
- 用户收藏表。

### 3.4 验收标准

- 用户可以注册、登录、退出。
- 登录后刷新页面仍保持登录状态。
- 未登录用户可以继续浏览基础内容。
- 登录用户保存的数据能归属到当前用户。
- 移动端可以使用同一个账号体系读取用户数据。

## 4. 阶段二：当前必须完善的基础能力

优先级：P0  
阶段目标：先把现有主流程补到可靠，避免在不稳定基础上叠创新功能。

### 4.1 音标基础完善

必须完成：

- 校对所有音标发音技巧。
- 校对示例词。
- 校对词首、词中、收音、对比音说明。
- 补齐不存在对比音或不存在词尾音时的说明。
- 明确静态音频、TTS、浏览器朗读的优先级。
- 移动端音标内容与 Web 保持一致。
- 移动端默认播放逻辑改为只播放 1 次音标，不默认播放词汇。

验收标准：

- Web 和移动端同一个音标展示的规则说明一致。
- 点击音标能正常播放。
- 点击示例词只播放该示例词。
- 不再出现默认播放音标和词汇多遍的移动端问题。

### 4.2 词汇和例句基础数据清洗

必须完成：

- 清洗重复词。
- 修正疑似错词。
- 补充词汇来源。
- 明确词汇是否来自平台内容或用户导入。
- 清洗例句翻译。
- 明确例句音频字段和播放规则。

验收标准：

- 平台词汇和用户词汇可以区分。
- 平台例句和用户例句可以区分。
- 错词和重复词不继续扩大。

### 4.3 教材 PDF 和 AI 问本页基础稳定

必须完成：

- 稳定 PDF 上传、转换、翻页、删除。
- AI 回答排版优化，避免 `###`、`**` 等 Markdown 标记污染页面。
- 明确 AI 问本页的数据范围：当前页、相邻页、必要时截图。
- AI 未配置时不影响教材阅读。

验收标准：

- 用户能上传 PDF 并稳定阅读。
- AI 问本页能基于当前页回答。
- AI 不声称读过整本 PDF。
- 未配置 AI key 时页面仍可阅读。

### 4.4 移动端内容对齐

必须完成：

- 音标首页 UI 按设计稿还原。
- 音标详情页按设计稿实现。
- 词汇页基础内容与 Web 一致。
- 教材页先保留正确的教材库和教材详情结构。
- 移除所有开发说明类文案。

验收标准：

- iPhone 和 iPad 都能自然展示核心页面。
- 移动端内容不明显落后于 Web。
- 移动端不展示“和 Web 保持一致”这类给开发者看的文案。

## 5. 阶段三：创新点一，音标与词汇联动

优先级：P1  
阶段目标：让音标不再孤立学习，而是和真实词汇训练产生关联。

### 5.1 音标关联词汇

核心能力：

- 每个音标关联一组词汇。
- 词汇用于展示音标在不同位置的实际发音。
- 音标详情中展示关联词汇。
- 用户点击关联词汇可播放词汇音频。

数据来源：

1. 平台内容。
2. 用户自己添加的关联词汇。
3. 用户导入的词汇资料，系统帮助生成词汇语音后，可沉淀为用户自己的音标关联词汇库。

依赖：

- 用户登录。
- 用户词汇表。
- 音标和词汇的关联表。
- 音频播放和 TTS 兜底。
- 用户导入内容解析和语音生成能力。

验收标准：

- 平台可配置每个音标的示范词。
- 登录用户可为音标添加自己的词汇。
- 用户添加的词汇只归属自己。
- 音标详情可以区分平台词汇和用户词汇。
- 用户导入词汇后，可以选择关联到一个或多个音标，并生成对应词汇语音。

### 5.2 音标和词汇的关联记忆

核心能力：

- 单词卡片中的音标可交互。
- 用户在学词时能看到该词包含哪些重点音标。
- 用户薄弱音标可以在词汇学习中被再次提醒。

价值：

- 用户学词时顺便复习音标。
- 发音训练不再只发生在音标页。

验收标准：

- 词汇详情能展示关联音标。
- 点击词汇中的音标可回到音标说明。
- 系统可以基于音标练习记录推荐含该音标的词汇。

## 6. 阶段四：创新点二，词汇与例句联动

优先级：P1  
阶段目标：让词汇不只是单词列表，而能进入例句、语音和复习。

### 6.1 词汇关联例句

核心能力：

- 每个词汇可关联平台例句。
- 用户可为词汇添加自己的例句。
- 用户可从教材中把句子添加到例句库，并关联到词汇。

词汇数据来源：

1. 平台内容。
2. 用户导入。
3. 用户在音标学习、教材学习或 AI 对话训练中保存的词汇。

例句数据来源：

1. 平台内容。
2. 用户自己导入。
3. 教材库句子添加到例句库。
4. 用户围绕词汇自行编写或通过 AI 辅助生成的例句。

依赖：

- 用户登录。
- 用户词汇表。
- 用户例句表。
- 词汇和例句关联表。
- 教材句子抽取能力。
- 用户导入内容解析能力。
- 用户私有音频资源表。

验收标准：

- 词汇详情能看到关联例句。
- 用户能给词汇新增例句。
- 用户能从教材页选中句子并添加到例句库。
- 平台例句和用户例句可以区分。
- 用户导入词汇或例句后，系统可以为韩语文本生成语音，并保存到用户私人词汇/例句库。

### 6.2 例句语音生成

核心能力：

- 用户导入例句后，系统支持生成对应韩语语音。
- 生成后的音频可以保存到用户例句库。
- 用户可用该音频进行听读和跟读。
- 用户导入的词汇也应支持生成词汇语音，避免私人词汇库只有文本没有听力训练能力。

依赖：

- TTS 服务。
- 生成音频存储策略。
- 用户例句表。
- 用户音频资源表。

验收标准：

- 用户添加例句后可以生成语音。
- 生成的语音能在例句详情中播放。
- 音频归属当前用户。
- 用户添加词汇后可以生成语音，并在词汇详情中播放。

### 6.3 用户私人学习资料库

核心能力：

- 用户可以导入自己的词汇资料和例句资料。
- 系统帮助识别韩语文本、中文/母语释义、来源、分类和关联关系。
- 系统为用户导入的韩语词汇和例句生成语音。
- 用户可以维护自己的私人词汇库、私人例句库和音标关联词汇库。
- 平台内容和用户内容在界面上可区分，但学习时可以统一进入听、读、跟读、复习流程。

数据来源：

1. 平台预置内容。
2. 用户手动新增。
3. 用户批量导入。
4. 用户从教材页保存。
5. 用户从 AI 生成内容中确认保存。

验收标准：

- 用户导入资料不会污染平台公共内容。
- 用户导入词汇和例句都可以生成语音。
- 用户可以删除、编辑、重新生成自己的语音资源。
- 私人词汇库和私人例句库支持跨 Web、iPhone、iPad 同步。
- 未登录用户不能被承诺拥有跨设备私人库。

## 7. 阶段五：创新点三，教材增强 AI 场景

优先级：P1  
阶段目标：让教材从静态 PDF 变成可问、可听、可练、可沉淀的学习空间。

### 7.1 AI 问本页

当前状态：

- Web 已有基础能力。

需要增强：

- 优化 AI 面板 UI。
- 优化回答排版。
- 支持本页语法、生词、音变规则、专项训练、课外主题扩展。
- 明确视觉模型优先使用 Gemini 2.5 Flash。

验收标准：

- 用户点击教材页 AI 入口可以围绕当前页提问。
- AI 回答能根据当前页内容组织。
- AI 可生成本页相关练习。
- AI 不脱离当前页乱讲。

### 7.2 教材 AI 语音对话训练入口

核心能力：

- 用户可以从教材页进入 AI 语音对话训练。
- 训练内容可基于当前教材主题。
- 用户也可以自定义主题。
- 支持不同 AI 人声音色。
- 支持不同语速。

价值：

- 用户学完教材词汇和语法后，直接进入听说训练。
- 补齐读写之外的听说训练。

依赖：

- 用户登录。
- AI 对话服务。
- 语音识别。
- 语音合成。
- 对话历史。
- 教材页上下文。

验收标准：

- 用户可从教材当前页发起语音对话训练。
- AI 对话主题可基于教材页内容生成。
- 用户可选择不同音色和语速。
- 对话记录归属当前用户。

### 7.3 教材文本转语音并添加到例句库

核心能力：

- 用户在教材页选择文本。
- 系统将所选文本转换成语音。
- 用户可以播放生成音频。
- 用户可以把文本和音频一起添加到例句库。

数据来源：

- 用户自己导入的 PDF 教材。
- 教材内选中的句子。
- 用户自己上传或映射的教材音频。

依赖：

- PDF 文本选择。
- TTS 生成。
- 用户例句库。
- 用户音频资源表。
- 教材和例句的关联关系。

验收标准：

- 用户能从教材页选中句子。
- 用户能生成该句子的语音。
- 用户能把句子和音频保存到例句库。
- 保存后可在例句模块看到该句子。

## 8. 阶段六：创新点四，语音评分与发音反馈

优先级：P1/P2  
阶段目标：让用户知道自己哪里读得不准，而不是只听标准音。

### 8.1 词汇语音评分，音标级识别

核心能力：

- 用户朗读词汇。
- 系统识别用户发音。
- 系统反馈到音标级别，而不是只给总分。

价值：

- 帮助用户知道具体哪个音标发错。
- 将词汇训练和音标训练打通。

依赖：

- 录音。
- STT 或发音评测服务。
- 音标级对齐。
- 音标和词汇映射。

验收标准：

- 用户读一个词后能得到反馈。
- 反馈指出具体音标问题。
- 用户可跳回对应音标学习页。

### 8.2 例句语音评分

核心能力：

- 用户朗读例句。
- 系统反馈句子层面的发音、节奏或流利度。
- 重点句子可反复跟读。

依赖：

- 用户例句库。
- 录音。
- 语音评测。
- 评分历史。

验收标准：

- 用户能录制例句跟读。
- 系统能给出可解释反馈。
- 评分结果可以回看。

### 8.3 发音锚点纠错

核心能力：

- 当用户某个词或句子读不好时，系统识别问题音标。
- 系统推荐用户先练对应音标。
- 用户完成音标练习后再回到词汇或例句。

验收标准：

- 评分结果能关联到音标。
- 用户能从评分反馈跳转到音标详情。
- 音标练习完成后能回到原训练任务。

## 9. 阶段七：跨端同步与 iPad 教材工作台

优先级：P2  
阶段目标：让 Web、iPhone、iPad 共享同一套学习数据，并发挥各端优势。

### 9.1 跨端同步

同步内容：

- 账号。
- 教材库。
- 学习进度。
- 词汇状态。
- 音标练习记录。
- 例句库。
- 收藏。
- 笔记。
- 批注。
- AI 历史。

验收标准：

- Web 添加的词汇，iPhone 可以看到。
- iPad 阅读到某页，Web 可以继续。
- 教材页添加的例句，移动端例句库可以看到。

### 9.2 iPad 教材工作台

核心能力：

- 大屏 PDF 阅读。
- Apple Pencil 批注。
- 划线和备注。
- 左侧教材，右侧音频、词汇、笔记。
- 点击句子播放音频。
- 听力材料分段播放。
- 教材页 AI 问答。

验收标准：

- 用户可以在 iPad 完成一页教材的阅读、听音频、批注、问 AI。
- iPad 上产生的笔记和例句可同步到 Web 和 iPhone。

## 10. 阶段八：国际化与全局能力增强

优先级：P2/P3

### 10.1 全球多语言版本

核心定位：

- Korean Learn 面向全球不同母语国家的用户学习韩语。
- 当前中文版本是第一阶段测试版本，先验证产品形态、训练闭环、内容结构和 AI 教材学习能力。
- 中文版本稳定后，再推出多语言版本。

核心能力：

- 用户可选择界面语言。
- 用户可选择解释语言和翻译语言。
- 音标发音技巧支持不同母语版本。
- 词汇释义支持多语言。
- 例句翻译支持多语言。
- 教材 AI 问本页支持按用户母语回答。
- AI 语音对话训练可根据用户母语背景给出发音提醒。

建议优先语言：

- 英文。
- 日文。
- 越南语。
- 泰语。
- 西班牙语。

依赖：

- 用户语言偏好。
- 多语言文案资源。
- 可维护的内容翻译机制。
- AI 回答语言控制。
- 音标和语法解释的多语言版本。

验收标准：

- 用户可以切换界面语言。
- 同一词汇可以展示不同语言释义。
- 教材页 AI 可以按用户选择的母语回答。
- 音标详情不再只提供中文解释。

### 10.2 多人声、多语速训练

核心能力：

- 语音生成支持不同人声。
- AI 语音对话支持不同人声。
- 听力和口语训练支持不同语速。

价值：

- 避免用户只适应单一音色。
- 提升真实听力和口语训练效果。

适用模块：

- 音标。
- 词汇。
- 例句。
- 教材 AI 语音对话。
- 教材文本转语音。

验收标准：

- 用户可以选择音色。
- 用户可以选择语速。
- 同一文本可用不同声音训练。
