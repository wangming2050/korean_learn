# Korean Learn 产品需求文档

更新日期：2026-06-23  
文档版本：v2，业务规则型 PRD  
适用范围：当前 Web 主产品、移动端原型、后续 iPad 方向、未来多语言版本

## 1. 文档定位

本文档不是产品介绍文，也不是愿景文档。本文档用于描述 Korean Learn 当前真实业务功能、页面规则、用户操作、系统响应、数据来源、异常状态和验收标准。

后续设计、开发、测试、重构和 Codex 执行任务时，应优先参考本文档判断“这个功能应该怎么表现”。

### 1.1 状态标记

本文档使用以下状态：

- `已实现`：当前代码中已经存在，可在 Web 或移动端验证。
- `原型中`：界面或部分逻辑已经存在，但数据、状态或交互还不完整。
- `需补齐`：产品上需要，但当前实现不足。
- `未来规划`：当前版本不做，但属于后续明确方向。

## 2. 产品背景

Korean Learn 长期面向全球不同母语的韩语初学者，解决的是入门阶段“看资料多，真正训练少”的问题。当前版本优先服务中文母语用户，用中文版本验证产品方向、学习流程、内容组织和训练闭环；当中文版本稳定后，产品将扩展为多语言版本。

当前产品重点不是提供大量课程，而是围绕韩语学习里的小单元建立训练闭环：

- 音标：看见字母，知道怎么发音，能听和跟读。
- 词汇：看到韩文，理解中文，能逐步进入复习。
- 例句：在场景中理解句子，能听和读。
- 教材：围绕 PDF 当前页阅读、提问、理解。
- AI：帮助解释当前页内容，不替代教材和用户判断。

## 3. 当前版本目标

### 3.1 当前版本要做到

- Web 端可稳定使用音标、词汇、例句、教材 PDF 和 AI 助教。
- 音标页面要从“资料展示”变成“发音训练入口”。
- 教材页面支持用户本地上传 PDF，并围绕当前页向 AI 提问。
- 移动端先和 Web 的核心内容保持一致，不优先做复杂个性化。
- iPhone 和 iPad 使用同一套移动端项目，按屏幕做适配。

### 3.2 当前版本不做

- 不做课程售卖。
- 不做班级、老师、作业、机构后台。
- 不做社区内容流。
- 不做没有登录态支撑的真实学习计划。
- 不默认提供未经授权的内置教材资源。
- 不把 TTS 当作最终标准发音素材。
- 不在当前阶段一次性完成全球所有语言版本。

## 4. 用户角色

### 4.1 学习用户

状态：`已实现`

当前学习用户无需登录即可使用 Web 学习功能。

可做：

- 查看学习路径。
- 学习音标。
- 查看词汇。
- 查看例句。
- 上传本地 PDF 教材。
- 对 PDF 当前页提问。
- 切换明暗主题。

限制：

- 没有跨设备学习记录。
- 没有真实个人词汇掌握状态。
- 本地 PDF 只保存在当前浏览器。
- 当前界面和学习解释主要为中文，尚未支持多语言切换。

### 4.2 管理员

状态：`已实现`

管理员通过 `/admin/login` 登录。

可做：

- 进入场景管理。
- 进入句子管理。
- 新增、编辑、删除场景。
- 新增、编辑、删除句子。

限制：

- 当前没有词汇后台。
- 当前没有音标内容后台。
- 当前旧教材资料后台不作为主入口开放。

### 4.3 登录用户

状态：`未来规划`

后续登录用户用于支撑：

- 多端同步。
- 学习进度。
- 词汇掌握状态。
- 教材库。
- 笔记。
- 批注。
- 收藏。
- AI 历史。

当前版本没有此角色。

## 6. 全局业务规则

### 6.1 登录与权限

状态：`已实现`

规则：

- 普通学习页面不要求登录。
- 管理后台页面要求管理员登录。
- POST、PUT、DELETE 类型的数据写入接口默认要求管理员登录。
- 公开 POST 接口例外：
  - `/api/admin/login`
  - `/api/admin/logout`
  - `/api/pdf-assistant/chat`
  - `/api/pdf-assistant/extract-toc`
  - `/api/tts/synthesize`

验收标准：

- 未登录访问 `/admin/scene` 或 `/admin/sentence` 时，应跳转登录页。
- 未登录调用受保护写入接口时，应返回未登录提示。
- 用户端 GET 学习数据不应被后台登录状态阻塞。

### 6.2 主题切换

状态：`已实现`

规则：

- 用户点击顶部明暗主题按钮切换主题。
- 主题状态保存在浏览器本地。
- 页面刷新后应保留上次主题。
- 按钮使用图标，不使用文字描述。

验收标准：

- 明暗主题切换不影响当前所在页面。
- 切换主题后音标、词汇、教材等主要页面仍可阅读。

### 6.3 音频播放通用规则

状态：`已实现 / 需补齐`

通用规则：

- 同一时间只应有一个主要学习音频播放。
- 用户触发新音频时，应停止或中断旧播放队列。
- 静态音频优先于 TTS。
- TTS 优先于浏览器朗读。
- 浏览器朗读只是兜底，不作为标准学习音频。

推荐优先级：

```text
静态 mp3
  -> 服务端 TTS
  -> 浏览器 SpeechSynthesis
  -> 展示不可播放提示
```

验收标准：

- 点击新的音标或示例词时，旧音频不应继续叠加播放。
- TTS 未配置时，页面不应白屏。
- 浏览器朗读不可用时，应给用户可理解的提示。

### 6.4 数据来源规则

状态：`已实现 / 原型中`

当前数据来源：

- MySQL：场景、句子、词汇、旧教材资料表。
- JS 常量：音标详情、发音说明、音标分组。
- 静态文件：音标音频、词汇音频、前端资源。
- IndexedDB：用户本地 PDF、页图、文本、AI 历史。
- localStorage：主题和部分界面状态。
- 移动端本地兜底：Web API 不可用时的默认数据。

业务要求：

- 用户可感知的数据应尽量来自统一来源。
- Web 和移动端不应长期维护两份不一致的音标详情。
- 用户个人数据在没有账号前只能算本地数据或临时数据。

## 7. 功能需求详述

## 7.1 学习路径

状态：`已实现`

### 功能目标

给新用户一个学习入口和使用方向，不替用户生成强制学习计划。

### 用户入口

用户打开首页后进入学习路径页，或点击顶部“学习路径”。

### 页面内容

学习路径页应提供：

- 四十音入门入口。
- 词汇入口。
- 例句入口。
- 教材入口。
- 网站使用思路或学习方法提示。

### 用户操作与系统响应

1. 用户点击某个学习路径卡片。
   - 系统切换到对应模块页面。
   - 不创建学习计划。
   - 不保存个人进度。

2. 用户切换到其他顶部导航。
   - 系统显示对应页面。
   - 当前没有登录态，不展示个人进度恢复。

### 数据来源

- 页面静态内容。
- 顶部导航状态。

### 当前限制

- 学习路径只是引导，不是真实个性化学习计划。
- 没有用户进度和完成状态。

### 验收标准

- 首次访问能看到学习入口。
- 点击入口能进入对应模块。
- 页面不出现“系统已经为你制定计划”这类不真实文案。

## 7.2 音标模块

状态：`已实现 / 持续优化`

### 功能目标

帮助用户学习韩语四十音和收音规则，重点是听、读、辨别，而不是只展示字母表。

### 用户入口

用户点击顶部“音标”进入。

### 内容结构

音标内容按学习结构展示：

- 辅音。
  - 松音。
  - 紧音。
  - 送气音。
- 元音。
  - 单元音。
  - 双元音。
- 收音。
  - 基础收音。
  - 双收音。

### 默认状态

Web：

- 默认展示音标分组。
- 详情区默认隐藏。
- 默认勾选“音标”。
- 默认不勾选“示例单词”。
- 播放次数默认 1。
- 不自动播放，直到用户点击音标。

移动端目标规则：

- 默认播放 1 次音标。
- 默认不自动读示例词。
- 示例词应由用户进入详情后主动点击播放。

### 页面元素

Web 音标页包含：

- 播放选项：音标。
- 播放选项：示例单词。
- 播放次数输入。
- 音标分组列表。
- 音标卡片。
- 音标详情区。

音标详情区应包含：

- 当前音标。
- 罗马音或音名。
- 发音技巧。
- 位置说明。
- 示例词。
- 对比词。
- 当前规则说明。
- 可点击播放的示例词。

### 用户操作与系统响应

#### 操作 1：点击音标卡片

系统响应：

- 设置当前音标为选中状态。
- 展开该音标详情。
- 停止之前正在播放的队列。
- 根据当前播放选项生成新播放队列。
- 立即开始播放。

特殊规则：

- 如果用户点击的是当前已经选中的音标，则关闭详情区。
- 切换到另一个音标时，旧音频应停止，新音标成为选中状态。

#### 操作 2：选择“音标”

系统响应：

- 后续点击音标时，队列包含音标本身。
- 音标播放文本优先使用 `playbackText`，其次使用音标或音名。

#### 操作 3：选择“示例单词”

系统响应：

- 后续点击音标时，队列包含当前音标详情里的示例词。
- 示例词应包含详情页中的示范词，而不是只读卡片上的单个词。

#### 操作 4：同时选择“音标”和“示例单词”

系统响应：

- 播放顺序为：先音标，再示例词。
- 每个播放项按播放次数重复。
- 当前实现中每个播放项之间有短暂停顿，给用户跟读时间。

示例：

```text
播放次数 = 2
选择音标 + 示例单词

播放 ㄱ 2 次
停顿
播放 가방 2 次
停顿
播放 고기 2 次
...
```

#### 操作 5：修改播放次数

系统响应：

- 播放次数必须大于等于 1。
- 非法输入自动回到 1。
- 之后的新播放队列使用新的播放次数。

#### 操作 6：点击详情页示例词

系统响应：

- 停止当前音标播放队列。
- 只播放被点击的示例词。
- 播放次数使用全局播放次数。
- 不自动继续播放其他示例词。

### 播放优先级

音标本身：

```text
letter_audio_url
  -> /api/tts/synthesize
  -> 浏览器韩语朗读
```

示例词：

```text
word-audio-map.js 映射音频或 example.audioUrl
  -> /api/tts/synthesize
  -> 浏览器韩语朗读
```

### 数据来源

后端：

- `/api/letters`
- `handler/sentence.py` 中的 `KOREAN_LETTERS`

前端：

- `static/js/app.js` 中的 `LETTER_DETAILS`
- `static/js/app.js` 中的 `PHONETIC_SECTIONS`
- `static/js/word-audio-map.js`

移动端：

- `mobile/src/api/webData.ts`
- Web API `/api/letters`
- 移动端本地 `LETTER_DETAILS` 和 `PHONETIC_SECTIONS`

### 空状态

如果 `/api/letters` 返回空：

- 页面应展示音标数据不可用提示。
- 不应渲染空白区域。

当前状态：

- 需要进一步补齐明确空状态 UI。

### 异常状态

- 音频文件不存在：降级到 TTS。
- TTS 未配置：降级到浏览器朗读。
- 浏览器朗读失败：提示无法播放。
- 用户未选择音标和示例单词：提示至少选择一个播放内容。

### 当前风险

- Web 和移动端音标详情重复维护，容易不一致。
- TTS 发音质量不能保证教学准确性。
- 示例词、对比词和收音说明需要人工校对。

### 验收标准

- 用户点击任一音标后，能看到详情。
- 点击不同音标时，选中态和详情内容同步变化。
- 音标播放优先使用静态音频。
- 示例词可单独点击播放。
- 播放次数为 2 时，同一播放项应播放 2 次。
- 不勾选任何播放内容时，系统给出明确提示。
- 移动端默认不应自动播放“音标 + 词汇 2 遍”。

## 7.3 词汇模块

状态：`已实现 / 原型中`

### 功能目标

让用户浏览和复习基础韩语词汇，为后续教材学习和句子理解做准备。

### 用户入口

用户点击顶部“词汇”进入。

### 当前页面能力

Web：

- 通过 React 组件展示词汇页面。
- 从 `/api/vocabulary` 获取词汇。
- 词汇包含韩文、中文、分类或来源字段。

移动端：

- 通过 `fetchWebLearningData()` 获取 `/api/vocabulary`。
- Web API 不可用时，移动端词汇列表可能为空或使用兜底数据。

### 用户操作与系统响应

#### 操作 1：进入词汇页

系统响应：

- 请求词汇数据。
- 渲染词汇列表或词汇学习界面。
- 如果请求失败，应显示数据不可用或兜底内容。

#### 操作 2：搜索或筛选词汇

当前接口支持：

- `/api/vocabulary?q=关键词`

系统响应：

- 根据关键词返回匹配词汇。

当前限制：

- 词汇页的学习状态不是用户真实状态。
- 没有账号体系，所以“未学习、待复习、已掌握”不能跨设备保存。

### 数据来源

- `/api/vocabulary`
- `handler/sentence.py`
- `handler/seed_content.py`
- `sql/schema.sql` 中的 `vocabulary` 表和种子数据。

### 空状态

应展示：

- 未找到词汇。
- 数据加载失败。
- 暂无词汇数据。

### 异常状态

- 数据库不可用：应使用兜底或展示错误。
- 搜索无结果：不应白屏。

### 当前风险

- 词汇种子数据存在疑似错词和重复词。
- 学习状态尚未接入真实用户数据。

### 验收标准

- 用户进入词汇页可以看到词汇内容。
- 搜索无结果时页面有明确提示。
- 词汇状态不应被文案包装成已经真实同步的个人记录。

## 7.4 例句模块

状态：`已实现 / 原型中`

### 功能目标

让用户在具体生活场景中学习基础韩语句子，支持理解、听音和跟读方向扩展。

### 用户入口

用户点击顶部“例句”进入。

### 内容结构

当前句子按场景组织：

- 学校。
- 医院。
- 交通。
- 餐厅。
- 购物。
- 住宿。

每条句子包含：

- 韩文。
- 中文。
- 情境。
- 音频地址。
- 音频开始时间。
- 音频结束时间。
- 场景 ID。

### 用户操作与系统响应

#### 操作 1：进入例句页

系统响应：

- 请求 `/api/scenes`。
- 请求 `/api/sentences`。
- 按场景组织句子。
- 渲染场景和句子列表。

#### 操作 2：选择场景

系统响应：

- 展示该场景下的句子。
- 不改变数据库。

#### 操作 3：播放句子

系统响应：

- 如果句子有音频地址，优先播放音频。
- 如果配置了开始和结束时间，应按时间段播放。
- 如果没有音频，可使用浏览器朗读兜底。

### 数据来源

- `/api/scenes`
- `/api/sentences`
- `scene` 表。
- `sentence` 表。
- 默认兜底句子 `DEFAULT_SENTENCES`。

### 异常状态

- 数据库查询失败时，后端可返回兜底句子。
- 没有场景或句子时，应展示空状态。
- 音频缺失时，不应阻塞句子展示。

### 当前限制

- 跟读、录音、评分不是 Web 当前稳定主能力。
- 移动端有录音和跟读相关原型，但还不是完整产品闭环。

### 验收标准

- 用户能看到场景列表。
- 用户能看到场景下的韩文和中文句子。
- 没有音频时句子仍可阅读。
- 后台新增句子后，用户端能读取到新句子。

## 7.5 教材 PDF 模块

状态：`已实现 / 需增强`

### 功能目标

让用户上传自己的韩语教材 PDF，在浏览器本地阅读，并围绕当前页进行理解和提问。

### 用户入口

用户点击顶部“教材”进入。

### 核心原则

- 当前版本不默认分发公共内置教材。
- 用户 PDF 不上传服务器。
- PDF 在浏览器本地解析。
- 页图、文本、目录和 AI 历史保存在 IndexedDB。

### 默认状态

当用户没有上传教材：

- 显示本地教材库空状态。
- 提供“上传 PDF”入口。
- 不直接进入教材详情页。

当用户已上传教材：

- 显示本地教材库。
- 用户可选择某本教材继续阅读。
- 教材状态可能是转换中、已就绪或失败。

### 上传规则

用户上传 PDF 时：

- 只能选择 PDF 文件。
- 单文件最大 300MB。
- 最大页数 600 页。
- 文件名默认作为教材标题。
- 系统创建本地教材记录。
- 系统开始在浏览器本地转换 PDF。

系统处理：

```text
PDF 文件
  -> PDF.js 加载
  -> 提取页数
  -> 提取目录
  -> 提取文本
  -> 生成缩略图和高清页图
  -> 保存到 IndexedDB
  -> 更新教材状态
```

### 阅读器规则

教材阅读器包含：

- 页码输入。
- 当前页总页数展示。
- 上一页按钮。
- 下一页按钮。
- 页面预览图。
- 加载骨架状态。
- 目录或章节入口。
- AI 助教面板。

用户操作：

1. 输入页码。
   - 系统跳转到对应页。
   - 页码不得小于 1。
   - 页码不得大于总页数。

2. 点击上一页。
   - 当前页大于 1 时跳转上一页。
   - 当前页为 1 时按钮不可用。

3. 点击下一页。
   - 当前页小于总页数时跳转下一页。
   - 当前页为最后一页时按钮不可用。

4. 切换目录。
   - 系统跳转到目录对应页。

### 目录识别规则

系统按以下顺序识别目录：

```text
PDF 内置目录
  -> 扫描前 30 页文本
  -> 文本不可用时，对候选目录页截图调用 Gemini
  -> 无法识别时展示无目录状态
```

当前常量：

- 文本扫描最多 80 页。
- 目录扫描最多 30 页。
- 目录解析候选最多 8 页。

### 删除规则

用户删除本地教材时，应删除：

- 教材记录。
- 页面图缓存。
- 页面文本缓存。
- 相关本地 AI 历史。

### 空状态

无教材：

- 展示“还没有本地教材，上传 PDF 后会在这里出现”类似提示。
- 提供上传入口。

无目录：

- 展示目录不可用或未识别提示。
- 不阻塞页码阅读。

### 异常状态

- 非 PDF 文件：提示选择 PDF。
- 文件超过 300MB：提示文件过大。
- 页数超过 600：提示页数超限。
- PDF 源文件不可用：提示重新上传。
- PDF 转换失败：教材状态标记失败，并允许重新上传。
- IndexedDB 不可用：提示当前浏览器不支持本地教材缓存。

### 当前限制

- PDF 和缓存只在当前浏览器。
- 换设备后无法继续阅读同一本本地 PDF。
- iPad 批注、划线、手写能力尚未产品化。

### 验收标准

- 用户能上传合法 PDF。
- 上传后能看到教材出现在教材库。
- 转换完成后能打开并翻页。
- 输入页码能跳转。
- 删除教材后，该教材不再出现在本地教材库。
- PDF 不应上传到服务器。

## 7.6 教材 AI 助教

状态：`已实现 / 需增强`

### 功能目标

让用户围绕当前 PDF 页面提问，获得语法、生词、句子和练习建议，降低教材理解成本。

### 用户入口

教材阅读器右侧 AI 助教面板。

### 页面元素

AI 助教应包含：

- 当前参考页提示。
- 历史记录区域。
- 快捷提问按钮。
- 问题输入框。
- 发送按钮。
- 展开或收起能力。

当前快捷提问包括：

- 讲解本页语法点。
- 生词解释。
- 根据本页出练习题。

### 用户操作与系统响应

#### 操作 1：用户输入问题并发送

系统响应：

- 如果没有打开教材，不发送请求。
- 如果问题为空，不发送请求。
- 显示用户问题。
- 显示“正在阅读当前页”状态。
- 收集当前页和相邻页文本。
- 判断当前页文本是否太少或乱码。
- 必要时截取当前页截图。
- 调用 `/api/pdf-assistant/chat`。
- 返回回答后渲染到历史中。

#### 操作 2：用户点击快捷提问

系统响应：

- 将快捷问题作为当前问题或直接触发提问。
- 仍然遵守当前页上下文规则。

#### 操作 3：用户查看历史

系统响应：

- 可按当前页或全部历史查看。
- 用户可删除单条历史。
- 用户可清空历史。
- 点击历史页码可跳转到对应页面。

### AI 上下文规则

请求内容包括：

- 用户问题。
- 文档标题。
- 当前页码。
- 总页数。
- 当前页和相邻页文本。
- 必要时的当前页截图。

截图触发条件：

- 当前页文本过短。
- 当前页文本疑似乱码。

### Provider 规则

当前代码支持：

- Gemini。
- OpenAI。

默认：

- `AI_PROVIDER=gemini`
- `GEMINI_MODEL=gemini-2.5-flash`

当前不支持：

- DeepSeek provider。

产品建议：

- 教材页视觉问答默认走 Gemini。
- DeepSeek 后续可作为纯文本问答 provider 接入。

### 异常状态

- AI key 未配置：返回“AI 助教未配置”类提示。
- 问题为空：提示问题不能为空。
- 问题过长：提示缩短问题。
- 模型请求失败：展示 AI 暂时不可用。
- 截图超限或格式不合法：丢弃截图，只用文本。

### 隐私规则

需要明确告知用户：

- 整本 PDF 不上传服务器。
- 提问时当前页和相邻页文本会发送给服务端。
- 文本不足时当前页截图会发送给模型服务。
- 不建议上传或询问敏感资料。

### 验收标准

- 用户在教材当前页提问后能获得回答。
- AI 回答不应声称看过整本 PDF。
- 未配置 AI key 时，教材阅读仍可使用。
- AI 历史能按页保存在当前教材记录下。
- 回答中的 Markdown 标记不应以混乱形式污染 UI。

## 7.7 TTS 与音频模块

状态：`已实现 / 需补齐素材`

### 功能目标

为音标、词汇、句子提供播放能力，并在静态音频缺失时提供兜底朗读。

### TTS 规则

接口：

- `/api/tts/synthesize`

请求限制：

- 文本不能为空。
- 单次文本不超过 500 字符。

返回：

- `audioUrl`
- `cached`
- `voice`
- `speakingRate`

缓存：

- 相同文本、声音、语速生成相同缓存文件。
- 缓存目录为 `static/generated-audio/`。

### 语速规则

- 普通速度：按传入或默认语速。
- 慢速：当前建议 0.75 左右。

### 异常状态

- 未配置 Google TTS key：返回 503。
- Google TTS 请求失败：返回错误。
- 没有返回音频：返回错误。

### 产品规则

- TTS 是兜底能力。
- 四十音标准发音不应长期依赖 TTS。
- 关键音标和示例词应逐步替换为真人标准音频。

### 验收标准

- TTS 配置正确时，用户可听到生成音频。
- 同一文本重复请求应优先返回缓存。
- TTS 不可用时，前端可降级到浏览器朗读。

- [ ] AI 未配置时教材阅读不阻塞。
- [ ] PDF 超过限制时有明确提示。
